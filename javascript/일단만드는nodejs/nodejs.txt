** nodejs

*** nodejs는 `자바사크립트 런타임`
런타임이란? 프로그래밍 언어가 구동되는 환경
기존엔 브라우저 내에서만 사용할 수 있었음 => nodejs 이후 서버 머신러닝, 데스크탑 앱까지 자바스크립트 구동되는 환경 확장 즉!! 자바스크립트를 이용할 수 있는 기반을 넓힘

*** 이벤트 기반
1. 실행 순서대로 호출스택에 쌓기
2. 백그라운드 필요 작업은 백그라운드로 이동
3. 백그라운드 작업 완료시 태스트 큐로
4. 다 완료되면 이벤트 루프가 테스트 큐 작업을 호출스택으로 이동

*** 논블로킹 I/O 
1. 이전 작업 기다리지 않고 다음 작업들 수행
2. 오래 걸리는 작업들은 백그라운드에서

*** 싱글 스레드
하나의 스레드에서 하나의 요청을 처리

**** var

**** let 
값을 재정의 할 수 있는 변수 선언 방식

**** const 
값을 재정의 할 수 없는 변수 선언 방식

*** 비구조화 할당
객체, 배열 안의 값을 추출해서 변수,상수에 바로 선언하는 문법

*** promise, async , await

*** 서버
네트워크를 통해 클라이언트에 정보나 서비스를 제공 컴퓨터 혹은프로그램
localhost  : 현재 컴퓨터 내부의 주소 (테스트용으로 많이 사용)
포트 : 서버내의 프로세스를 구분할 수 있는 번호


-----

*** npm
nodejs로 만들어진 패키지를 설치하고 관리해주는 툴
서로 의존되는 패키지들 존재
패키지의 버전마다 기능이 다를 수도 있음 => 문제점 => 설치한 패키지 버전을 관리해주는 파일 : package.json

*** 라우팅

---- 
*** http 메소드
서버에 요청을 보내는 방법
1. get : 요청 받은 정보를 검색하여 응답
2, post: 요청된 자원 생성
3. put :  요청된 자원 수정
4. delete :  요청된 자원 삭제
